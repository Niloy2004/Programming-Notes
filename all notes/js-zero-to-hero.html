01Introduction to JavaScript

JS kya hai aur kyu zaroori hai

JavaScript ek high-level, interpreted programming language hai jo originally sirf web pages ko interactive banane ke liye bani thi. Ab ye browser ke andar (frontend), server pe (Node.js), mobile apps mein (React Native), aur desktop apps mein (Electron — jaise tumne RO ERP banaya) — har jagah chalti hai.

JS browser mein kaise chalta hai?

  • HTML → structure banata hai (skeleton)
  • CSS → styling deta hai (looks)
  • JS → behavior aur interactivity deta hai (brain)
  • Browser ka JS Engine (Chrome mein V8) code ko parse karke run karta hai
index.html
<script>
  console.log("Hello Niloy!"); // browser console mein print hoga
</script>
Tip: Browser mein F12 daba ke Console tab khol — yahin JS practice karte hain sabse pehle.

02Variables

var, let, const — kab kaunsa use karna hai

Variables data store karne ke liye containers hain. JS mein 3 tarike hain declare karne ke:

KeywordReassign?Redeclare?Scope
varHaanHaanFunction scope (purana, avoid karo)
letHaanNahiBlock scope (modern default)
constNahiNahiBlock scope (fixed value)
variables.js
let name = "Niloy";
const college = "Mangalayatan University";
let semester = 7;

semester = 8; // ✅ let reassign ho sakta hai
// college = "XYZ"; ❌ Error — const reassign nahi hoga

Golden Rule: Default const use karo. Agar value change karni pade to let. var ko bhool jao — hoisting aur scope ke bugs deta hai.

03Data Types

JS mein data kis form mein store hota hai

Primitive Types (7)

  • String — "Niloy", text data
  • Number — 42, 3.14 (int aur float dono same type)
  • Boolean — true / false
  • Undefined — variable declare hua par value nahi di
  • Null — intentionally "koi value nahi"
  • BigInt — bahut badi numbers ke liye
  • Symbol — unique identifiers (rarely use hota hai)

Non-Primitive (Reference) Type

  • Object — arrays, functions, objects — sab isi ke andar aate hain
typeof.js
console.log(typeof "hello");   // "string"
console.log(typeof 25);       // "number"
console.log(typeof true);     // "boolean"
console.log(typeof undefined); // "undefined"
console.log(typeof null);      // "object" — famous JS bug!
console.log(typeof [1,2]);    // "object"
Interview favorite: typeof null "object" deta hai — ye JS ka 30 saal purana bug hai jo fix nahi kiya gaya (backward compatibility ke wajah se).

05Control Flow

if / else / switch — decisions kaise leta hai code

if-else.js
let marks = 72;

if (marks >= 90) {
  console.log("Grade: A+");
} else if (marks >= 70) {
  console.log("Grade: A");

  console.log("Grade: B");

switch.js
let day = 3;

switch (day) {
  case 1: console.log("Monday"); break;
  case 2: console.log("Tuesday"); break;

06Loops

Repeat karne wale kaam ke liye

loops.js
// for loop — jab exact count pata ho
for (let i = 1; i <= 5; i++) {
  console.log("Count: " + i);
}

// while loop — jab condition pe depend ho
let n = 0;
while (n < 3) {
  console.log("n is " + n);

  • for — jab loop count fixed ho
  • while — jab tak condition true rahe
  • for...of — arrays/strings ki values ke liye
  • for...in — object ki keys ke liye
  • break — loop ko turant rok do
  • continue — is iteration ko skip karke next pe jao

07Functions

Reusable code blocks — JS ka backbone

Function Declaration vs Expression vs Arrow

functions.js
// 1. Function Declaration — hoisted hota hai (upar bhi call kar sakte ho)
function greet(name) {
  return `Hello, ${name}!`;
}

// 2. Function Expression — hoisted nahi hota
const add = function(a, b) {
  return a + b;
};

// 3. Arrow Function — modern, short syntax
const multiply = (a, b) => a * b;

console.log(greet("Niloy"));
console.log(add(4, 5));
console.log(multiply(3, 4));

Default Parameters

default-params.js
function order(item, qty = 1) {
  return `${qty}x ${item}`;
}
console.log(order("Pizza")); // qty na di to default 1 use hoga
Arrow functions ka special behaviour: Inke pass apna this nahi hota — parent scope ka this use karte hain (Section 10 mein detail se dekhenge).

08Scope & Hoisting

Variable kahan accessible hai, aur JS use kaise "upar utha" leta hai

  • Global Scope — function ke bahar declare, kahin se bhi accessible
  • Function Scope — sirf usi function ke andar accessible (var)
  • Block Scope — { } ke andar accessible (let, const)
hoisting.js
console.log(x); // undefined (error nahi aayega — hoisted hai)
var x = 5;

// console.log(y); // ❌ ReferenceError — let/const "Temporal Dead Zone" mein hote hain
let y = 10;

console.log("x =", x, "y =", y);
Hoisting ka matlab: JS engine code run karne se pehle saare var aur function declarations ko file ke top pe "utha" leta hai (memory allocate karta hai). Isiliye function declaration ko use hone se pehle bhi call kar sakte ho, par let/const ke sath aisa nahi hota.

09Closures

JS ka sabse "interview-favorite" aur confusing topic

Closure ek aisi function hoti hai jo apne outer function ke variables ko yaad rakhti hai, chahe outer function execute ho chuka ho aur khatam ho gaya ho.

closures.js
function counter() {
  let count = 0; // ye "private" variable ban jata hai

  return function() {
    count++;
    return count;
  };
}

const increment = counter(); // counter() ek baar chala, khatam

console.log(increment()); // 1
console.log(increment()); // 2 — count yaad reh gaya!
console.log(increment()); // 3
Real use-case: Data privacy (private variables banana), function factories, aur event handlers mein state store karna — jaise tumne Simon Says game mein score track kiya hoga.

10The this Keyword

Context ke hisab se badalta hai — yahi confusion create karta hai

Contextthis kis se refer karta hai
Regular function (global call)undefined (strict mode) / window
Object methodWo object jispe method call hua
Arrow functionParent (enclosing) scope ka this
Constructor / ClassNaya bana object instance
this-keyword.js
const student = {
  name: "Niloy",
  greetNormal: function() {
    return `Hi, I'm ${this.name}`; // this = student object
  },
  greetArrow: () => {
    return `Hi, I'm ${this.name}`; // this = outer scope, name undefined!
  }
};

console.log(student.greetNormal()); // "Hi, I'm Niloy"
console.log(student.greetArrow());  // "Hi, I'm undefined"
Yaad rakhna: Object methods ke liye normal function use karo, arrow function nahi — warna this object ko refer nahi karega.

Important Array Methods (interview mein bahut puchte hain)

array-methods.js
const nums = [1, 2, 3, 4, 5];

const doubled = nums.map(n => n * 2);          // har element transform
const evens = nums.filter(n => n % 2 === 0); // condition ke hisab se chunta hai
const total = nums.reduce((acc, n) => acc + n, 0); // sab ko ek value mein combine
const found = nums.find(n => n > 3);          // pehla matching element

console.log("doubled:", doubled);
console.log("evens:", evens);
console.log("total:", total);
console.log("found:", found);
MethodKaam
map()Naya array banata hai, har element transform karke
filter()Condition true hone pe elements select karta hai
reduce()Poore array ko ek single value mein reduce karta hai
forEach()Har element pe loop chalata hai (kuch return nahi karta)
find()Pehla matching element return karta hai
findIndex()Pehla matching element ka index return karta hai
some()Kam se kam ek element condition pass kare to true
every()Sab elements condition pass karein tabhi true
sort()Array ko sort karta hai (original ko modify karta hai!)
slice()Part of array copy karta hai, original nahi badalta
splice()Elements add/remove karta hai, original ko modify karta hai

sort(), slice() aur splice() — inme confusion hota hai

sort-slice-splice.js
const scores = [40, 5, 100, 25];

// sort() default string ki tarah sort karta hai — numbers ke liye compare function do
const sorted = [...scores].sort((a, b) => a - b);
console.log("sorted:", sorted);

const letters = ["a", "b", "c", "d", "e"];
console.log(letters.slice(1, 3));  // ["b","c"] — copy, original safe

letters.splice(1, 2, "X"); // index 1 se 2 remove, "X" insert
console.log("after splice:", letters); // original badal gaya
Trap: sort() aur splice() original array ko modify karte hain (mutate). map(), filter(), slice() naya array banate hain, original safe rehta hai. Agar original nahi todna to spread [...arr] se copy bana lo pehle.

12Objects

Key-value pairs mein data — real world entities represent karne ke liye

objects.js
const project = {
  name: "RO ERP System",

    return `${this.name} built with ${this.stack.join(", ")}`;

console.log(project.name);        // dot notation

console.log(project.getSummary());

Bracket notation kab use karein: Jab key ka naam dynamic ho (variable mein store ho) ya usme space/special character ho. Warna dot notation zyada readable hai.

17Classes (OOP in JS)

Objects banane ka blueprint — real ERP jaisi apps mein useful

classes.js
class Product {
  constructor(name, price) {
    this.name = name;
    this.price = price;
  }

  getInfo() {
    return `${this.name} — ₹${this.price}`;
  }
}

// Inheritance
class DiscountedProduct extends Product {
  constructor(name, price, discount) {
    super(name, price); // parent constructor call
    this.discount = discount;
  }
  finalPrice() {
    return this.price - this.discount;
  }
}

const p1 = new Product("RO Filter", 1200);
const p2 = new DiscountedProduct("UV Lamp", 800, 100);

console.log(p1.getInfo());
console.log(p2.finalPrice());

Getters, Setters aur Static Methods

getters-static.js
class Circle {
  constructor(radius) {
    this.radius = radius;
  }

  get area() { // property jaise access hota hai, () ki zaroorat nahi
    return (Math.PI * this.radius ** 2).toFixed(2);
  }

  set diameter(d) { // value assign karne pe automatically chalta hai
    this.radius = d / 2;
  }

  static unitCircle() { // instance banaye bina call hota hai
    return new Circle(1);
  }
}

const c = new Circle(5);
console.log(c.area);       // get ki tarah, brackets nahi
c.diameter = 20;       // set ki tarah
console.log(c.radius);

const unit = Circle.unitCircle(); // class pe direct call, instance pe nahi
console.log(unit.radius);

21Promises

Async operation ka "future result" represent karta hai

Promise 3 states mein hota hai: pending (chal raha hai), fulfilled (success), rejected (fail).

promises.js
const orderPizza = new Promise((resolve, reject) => {
  const stockAvailable = true;

  setTimeout(() => {
    if (stockAvailable) {
      resolve("Pizza is ready! 🍕");
    } else {
      reject("Out of stock");
    }
  }, 1500);
});

orderPizza
  .then((result) => console.log(result))   // success
  .catch((err) => console.log(err))        // failure
  .finally(() => console.log("Order process done")); // hamesha chalega

22Async / Await

Promises ko likhne ka clean, synchronous-jaisa tareeka

async-await.js
function delay(ms) {
  return new Promise((resolve) => setTimeout(resolve, ms));
}

async function makeTea() {
  try {
    console.log("Boiling water...");
    await delay(1000); // yahan JS ruk jayega jab tak promise resolve na ho

    console.log("Adding tea leaves...");
    await delay(500);

    console.log("Tea is ready! ☕");
  } catch (error) {
    console.error("Something went wrong:", error);
  }
}

makeTea();
Yaad rakhna: await sirf async function ke andar hi kaam karta hai. Ye Promise ke resolve hone ka wait karta hai bina code ko block kiye (baaki app responsive rehta hai).

24Error Handling

Code crash na ho, gracefully handle ho

error-handling.js
function divide(a, b) {
  try {
    if (b === 0) {
      throw new Error("Cannot divide by zero!");
    }
    return a / b;
  } catch (error) {
    console.log("Error caught:", error.message);
    return null;
  } finally {
    console.log("Division attempt complete");
  }
}

console.log(divide(10, 2));
console.log(divide(5, 0));

26Map & Set

Objects aur Arrays ke alternative, kuch cases mein better

Map object jaisa hi hai (key-value pairs) but key koi bhi type ho sakti hai — sirf string nahi. Set array jaisa hai but sirf unique values store karta hai, duplicate automatically hat jate hain.

map-set.js
// Map
const userRoles = new Map();
userRoles.set("niloy", "admin");
userRoles.set("guest", "viewer");

console.log(userRoles.get("niloy"));
console.log(userRoles.size);
console.log(userRoles.has("guest"));

// Set
const tags = new Set(["js", "css", "js", "html"]);
console.log([...tags]); // duplicate "js" automatically hat gaya
Kab use karo: Duplicate hatane ho array se? [...new Set(array)] ek line mein kaam kar deta hai. Key-value store chahiye jisme key string na ho? Map use karo.

27Optional Chaining & Nullish Coalescing

Undefined/null errors se bachne ke liye — daily use hone wale operators

optional-chaining.js
const user = {
  name: "Niloy",
  address: { city: "Bankura" }
};

// purana tareeka — pehle check karna padta tha
// const pin = user.address && user.address.pincode;

// optional chaining (?.) — agar beech mein kuch undefined ho to crash nahi hoga
const pin = user?.address?.pincode;
console.log(pin); // undefined — error nahi aayega

// nullish coalescing (??) — sirf null/undefined pe fallback deta hai
const displayPin = pin ?? "Not provided";
console.log(displayPin);

// || vs ?? ka fark
let discount = 0;
console.log(discount || 10); // 10 — || 0 ko bhi falsy maanta hai
console.log(discount ?? 10); // 0  — ?? sirf null/undefined ko replace karta hai
Common bug: || se default value dete waqt agar actual value 0, "" ya false ho to wo bhi replace ho jayegi. Isliye asli "no value" check karne ke liye ?? use karo.
